# 效果评估体系（占位）

> 本目录用于存放 Agent 体系自身的运行效果评估。
> 当前为 V0.1 占位，后续可参照 `skills/` 仓库的 EVALUATION.md 建立 8 维度评估框架。

## 评估维度（建议）

| 维度 | 说明 |
|------|------|
| 闭环完整性 | 数据→模型→服务→商业 DAG 是否覆盖全部 15 角色 |
| 角色清晰度 | 每个角色职责/KPI/技能是否明确、无重叠 |
| 质疑有效性 | 质疑协议能否拦住自嗨（评测先行） |
| 可量化性 | KPI 是否都是具体数字 |
| 可安装性 | install.sh 能否一键部署到 .claude/ |
| 可初始化 | init.sh 能否写入模型公司信息 |
| 映射一致 | 每个 agent 的「可用技能」= skills/ 实际目录 |
| 文档一致 | README/WORKFLOW 与实际文件一致 |

## Sprint 归档

每次迭代复盘归档于此（benchmark + agent-outputs 子目录可按需创建）。
