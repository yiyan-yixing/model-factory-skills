# 数据质量校验

> 垃圾进，垃圾出。**数据质量是大模型效果的地基**——再好的超参也救不了脏数据。
> 这一页讲质量怎么验、门槛定什么、坏数据怎么拦。

## 质量校验三层

| 层 | 方法 | 抓什么 |
|----|------|--------|
| **统计层（自动）** | 分布/长度/重复/缺失 指标 | 整体异常 |
| **规则层（自动）** | 格式/编码/语言/长度 阈值 | 明显坏样本 |
| **采样层（人工）** | 抽样人工看 | 统计和规则抓不到的语义问题 |

> 自动层抓得快但有盲区，人工抽样抓语义但慢。**三层缺一不可**。

## 统计层该看什么

| 指标 | 异常信号 |
|------|----------|
| 长度分布 | 突然出现超长/超短尖峰 |
| 字段分布 | 某类别占比异常高（不平衡） |
| 重复率 | 高重复（见去重） |
| 缺失率 | 关键字段大量缺失 |
| 语言/编码分布 | 混入非目标语言/乱码 |
| 时间分布 | 数据时间偏斜（如全是某时期） |

> 统计层的价值：**让数据问题可见**。一张分布图比看一万条样本快。

## 规则层：定可执行的门槛

针对你的数据定**可机器校验**的规则：

```
- 长度 ∈ [min, max]
- 语言 = 目标语言（语言检测）
- 编码 = UTF-8 无乱码
- 不含 PII（正则/检测）
- 格式符合 schema（JSON 可解析 / 字段齐全）
- 不命中黑名单（有害/低质来源）
```

不满足 → 进隔离区人工复核或丢弃，**不直接喂训练**。

## 采样层：人工抽检

- 随机抽 N 条（如 100-500）人工过一遍
- 重点抽**边界和异常**（最长最短、低分、规则边缘的）
- 记录问题率，问题率高 → 整批退回重洗

> 自动校验全过的数据，人工一看"半数是废话/重复/错标"——这种情况很常见。人工抽样是最后一道真实验证。

## 质量门槛（收数据/喂训练前过）

| 门槛 | 经验值（需校准） |
|------|-----------------|
| 重复率 | ≤ 1-5% |
| 噪声/坏样本率 | ≤ 设定阈值 |
| 标注一致性（有标注数据） | ≥ 70-85% |
| 覆盖率 | 覆盖目标场景的核心 case |
| 格式合格率 | ≥ 95% |

> 门槛不是越严越好——太严数据量不够，太松质量差。在你的评测集上**反向校准**：哪档质量训出来的模型评测分最高。

## 数据质量的反模式

| 反模式 | 后果 |
|--------|------|
| 只信自动校验，不人工抽检 | 语义级问题漏网 |
| 不看分布直接训练 | 异常数据拉偏模型 |
| 门槛拍脑袋不定 | 质量随缘 |
| 坏数据默默丢弃不记录 | 不知道丢了多少、为什么 |
| 标注不一致也不校准 | 标签噪声，模型学不准 |

## 与其他角色的衔接

- @data-strategy 定质量标准 → 你执行校验，**不达标退回**
- @ml-trainer 收到的是**过质量门的数据**，否则有权拒收
- 质量报告归档（哪些批次、问题率、如何处理）

> 质量校验不是 @data-engineer 一个人闷头做——它是数据能不能进训练的**闸门**，和评测一样重要。
