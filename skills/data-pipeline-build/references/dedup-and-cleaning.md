# 去重与清洗

> 去重和清洗是数据流水线里**最被低估、但对效果影响最大**的环节。
> 重复数据让模型过拟+记忆；脏数据让模型学噪声。**宁可少而净，不要多而脏。**

## 去重：三层粒度

| 粒度 | 方法 | 抓什么 |
|------|------|--------|
| **精确去重** | hash 完全匹配 | 一模一样的副本 |
| **模糊去重** | MinHash / SimHash / 近似匹配 | 改了几个字的近似副本 |
| **语义去重** | 嵌入聚类 | 说同一件事的不同表述（高级，可选） |

> 大模型预训练数据**模糊去重是关键**——只去精确重复会漏掉大量"换了几个词的重复"，这些同样有害。

### 为什么去重重要（实证）

- 重复数据 → 模型**记忆**而非**泛化**（背下来而非学会）
- 重复 → **评测虚高**（训过的近似样本出现在评测里）
- 重复 → **训练效率低**（同样的信息训多遍）
- 高质量数据集（如各类公开预训练集）都把去重作为**核心步骤**

> 业界共识：**去重能显著提升模型质量**。这是少数有强证据的数据处理操作之一。

## 清洗规则（可配置、可复现）

清洗规则要写成**配置**，不写死脚本：

```
1. 格式归一化（空白/换行/编码）
2. 去除模板/导航/广告（网页数据）
3. 去毒 / 去有害
4. 脱敏（PII：邮箱/电话/身份证/银行卡）
5. 语言过滤（只留目标语言）
6. 质量过滤（长度/困惑度/启发式打分）
7. 去重（精确+模糊）
```

> 清洗规则版本化（见 versioning.md）——换数据集能复用，且能解释"这批数据怎么来的"。

## 去毒与安全清洗

- **有害内容过滤**：仇恨/暴力/色情等（分类器 + 规则）
- **PII 脱敏**：个人隐私必须脱敏，**合规底线**
- **低质来源过滤**：某些来源已知低质，整源剔除

> 去毒脱敏是**合规和安全底线**，不是可选项。上线后被发现训练数据含 PII/有害内容，是法律+信任双重事故。

## 常见清洗陷阱

| 陷阱 | 后果 | 对策 |
|------|------|------|
| 只做精确去重 | 大量近似重复漏网 | 上模糊去重（MinHash） |
| 清洗规则写死在脚本 | 不可复用不可复现 | 写成配置 |
| 去毒太狠 | 误删正常数据 | 阈值调参 + 抽检误删 |
| 不记录清洗过程 | 训练效果说不清 | 规则版本 + 处理日志 |
| 合成数据不清洗 | 合成垃圾污染 | 合成数据同样过质量门 |

## 合成数据的特殊清洗

合成数据（模型生成）要额外注意：
- **去重**：模型生成容易重复
- **质量过滤**：合成输出质量参差，要过滤
- **标注为合成**：与真实数据区分，便于追踪影响
- **防自循环**：别让模型学自己的输出退化

## 清洗的可观测性

每批数据清洗后产出报告：
- 输入多少 / 输出多少 / 各步丢弃多少
- 重复率 / 去毒率 / 脱敏数
- 清洗规则版本号

> 清洗不是黑盒。**能解释每一批数据怎么从原始变干净**，才能调试训练问题。
