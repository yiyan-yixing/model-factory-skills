# 项目上下文

> 此文件由所有 Agent 共享。描述公司/产品的基本信息，每个 Agent 都应该知道。

## 公司

- **名称**：待定
- **阶段**：0->1 创立期
- **模式**：大模型公司（用最少人数跑通 数据->模型->服务->商业 闭环）
- **方向**：垂直大模型能力 / 模型 API 服务
- **技术路线**：知识蒸馏（D7）— 从大模型定向蒸馏能力到 0.5B-3B 小模型

## 硬件约束

- **训练环境**：Mac 48G（Apple Silicon M 系列）
- **训练框架**：mlx-lm（首选），PyTorch+MPS（备选）
- **学生模型规模**：0.5B-3B（QLoRA 训练）
- **教师模型规模**：7B 本地 4-bit 推理 + 更强模型 API 调用
- **不能同时加载教师+学生**：需分阶段执行

## 产品

- **定位**：在某个垂直场景做出可本地部署、低延迟、低成本的蒸馏小模型能力
- **目标用户**：需要在端侧/本地运行垂直 AI 能力的开发者和产品团队
- **核心假设**：市场需要能在消费级硬件上运行的垂直 AI 能力，而非只有云 API 一种选择
- **差异化**：端侧可运行 + 垂直数据壁垒 + 零推理成本（本地 Mac 即服务器）

## 季度 OKR（2026 Q3）

- **O**：验证知识蒸馏路线在 Mac 48G 上可跑通，交付 2 个垂直蒸馏模型
- **KR1**：完成情感分类试点（SFT+评估+部署），验证 mlx-lm 训练流程（W3）
- **KR2**：验证蒸馏有效性—教师模型生成数据 vs 直接 SFT 的效果对比（W5）
- **KR3**：蒸馏流水线可复用—第二个垂直场景 3 天内完成从选场景到可调用 API（W8）
- **KR4**：补齐 DPO 对齐能力或确认 SFT 模拟方案可行（W10）

## 明确不做

- ❌ 通用大模型对标 GPT（打不过大厂）
- ❌ 自建超大规模 GPU 集群（烧不起）
- ❌ 堆人力标注（强自动化优先）
- ❌ AI 套壳（必须有数据/模型壁垒）
- ❌ ToC 消费级产品（先做 ToB/ToDev 变现路径短）
- ❌ 3B 以上学生模型训练（Mac 48G 不可行）
- ❌ 需要深度推理的垂直场景（小模型天花板）
- ❌ 自训基座（D6 生效：开源基座微调起步）

## 当前瓶颈

- 试点项目训练未启动（已有 PRD+数据+PyTorch 脚本，需迁移到 mlx-lm）
- DPO 对齐在 mlx-lm 上不支持，需评估替代方案
- 教师模型蒸馏 prompt 设计无经验积累
