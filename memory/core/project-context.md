# 项目上下文

> 此文件由所有 Agent 共享。描述公司/产品的基本信息，每个 Agent 都应该知道。

## 公司

- **名称**：待定
- **阶段**：0→1 创立期
- **模式**：大模型公司（用最少人数跑通 数据→模型→服务→商业 闭环）
- **方向**：垂直大模型能力 / 模型 API 服务

## 产品

- **定位**：在某个垂直场景做出可调用、可付费的模型能力（找到第一个付费场景）
- **目标用户**：待定（需通过可调用能力验证）
- **核心假设**：市场需要垂直场景的模型能力（不只是通用聊天），有人愿意为调用付费
- **差异化**：垂直数据壁垒 + 工程效率（15 人跑通闭环）

## 季度 OKR（2026 Q3）

- **O**：用可调用的模型能力赢得信任，获得第一批付费调用方
- **KR1**：构建垂直训练数据集 X 万条，质量达标（W4）
- **KR2**：模型在垂直评测基准达标，可调用（W8）
- **KR3**：API 上线，日调用 X 次（W10）
- **KR4**：付费调用方 3 个 / MRR ¥5,000（W12）

## 明确不做

- ❌ 通用大模型对标 GPT（15 人打不过大厂）
- ❌ 自建超大规模 GPU 集群（烧不起）
- ❌ 堆人力标注（强自动化优先）
- ❌ AI 套壳（必须有数据/模型壁垒）
- ❌ ToC 消费级产品（先做 ToB/ToDev 变现路径短）

## 当前瓶颈

- 验证 Problem-Solution Fit（垂直场景是否真有付费需求）
- 数据从哪来（垂直数据壁垒是核心）
- 算力成本可控
