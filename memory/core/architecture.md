```
数据层    ──→    模型层     ──→    服务层     ──→    商业层
采集/清洗        预训练/对齐       API/推理        增长/计费
@data-engineer   @ml-trainer      @backend        @growth
@data-strategy   @ml-alignment    @ml-serving     @devrel
                 @evaluation      @mlops          @ops
                                  @infra
```

- **数据层**：采集 → 清洗 → 版本化 → 质量校验 → 喂训练
- **模型层**：预训练 → 对齐 → 评测（闸门）→ 达标才进服务层
- **服务层**：推理优化 → 封装 API → 流水线自动化 → 部署监控
- **商业层**：开发者接入 → 用户增长 → AB/回滚 → 决策迭代

## ADR（Architecture Decision Records）

格式：`ADR-<编号> | <标题> | <状态>`

### ADR 模板

```
## ADR-<N>: <标题>

### 状态
提议 / 已采纳 / 已废弃 / 已替代

### 背景
为什么要做这个决策？

### 决策
我们选择了什么？

### 理由
为什么这么选？考虑了哪些替代方案？

### 后果
这个决策带来的好处和代价？
```
