## 大模型公司技术栈分层

| 层级 | 说明 | 典型选型 |
|------|------|----------|
| 数据 | 采集/清洗/版本/存储 | Python + SQL + DVC/版本化 + 对象存储 |
| 训练框架 | 预训练/微调 | PyTorch + DeepSpeed/Megatron + HF Transformers |
| 算力 | 训练/推理 GPU | 云 GPU（A100/H100）按需 / 托管集群 |
| 对齐 | SFT/RLHF | TRL / DeepSpeed-Chat / 自研 |
| 推理服务 | 推理引擎/量化 | vLLM / TGI / TensorRT-LLM + AWQ/GPTQ |
| 平台 | API/网关/数据库 | 网关 + PostgreSQL/向量库 + 对象存储 |
| 部署 | CI/CD/容器 | Docker + K8s/云托管 + GitHub Actions |
| MLOps | 流水线/实验追踪 | MLflow / W&B / 自研编排 |

## 选型原则

- **开源优先**：能用开源框架/基座就不从零造，社区大、出问题能搜到
- **算力成本可控**：训练/推理成本必须能支撑商业定价（推理成本 ≤ 定价 40%）
- **托管优先**：能用云托管/Serverless 就不自建集群
- **速度优先**：数据→模型→上线 < 1 周
- **可复现**：数据版本 + 超参 + 种子全记录
