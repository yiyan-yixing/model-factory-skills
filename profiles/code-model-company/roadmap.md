# 代码大模型公司路线图

> 从 0→1 跑通"代码数据→代码模型→IDE 服务→企业付费"的路线。
> 不是唯一路径，是**一条经过验证可行的主线**，按你的资源裁剪。

## 路线总览

```
[场景聚焦] → [基座+数据] → [训练+评测] → [IDE/API] → [企业私有化/Agent] → [商业化]
   决策         合规底线      防污染闸门      产品形态       差异化          变现
```

## 阶段 0：场景聚焦（W1-2）

**决策：先做哪个代码能力**

| 能力 | 难度 | 变现 | 建议 |
|------|------|------|------|
| 代码补全（FIM） | 低 | 红海 | 入门，但非差异化 |
| 代码生成 | 中 | 中 | 标配 |
| 代码 Agent（改bug/写测试） | 高 | 高 | **差异化重点** |
| 代码审查 | 中 | 中 | 企业有需求 |

> 建议：**补全打底（证明能跑）→ Agent 差异化（建壁垒）**。别只做补全，那是 Copilot 红海。

## 阶段 1：基座 + 数据（W2-4）

**关键决策 + 合规底线**

- **基座**：开源代码基座（StarCoder/CodeLlama/DeepSeek-Coder/Qwen-Coder 系）微调起步；自训代码基座留到有独占数据后
- **数据**：公开代码语料 + 去重 + 质量过滤
- **合规底线**：协议过滤 + 去凭证（见 `data-strategy.md`）—— **这一步不过不能往下走**

> 合规是代码大模型的生命线。@legal 必须介入协议白名单确认。

## 阶段 2：训练 + 评测（W4-7）

- 微调（SFT）→ 代码指令数据
- **评测闸门**：LiveCodeBench pass@1 达标 + 私有集
- 防污染：训练集剔除公开 benchmark
- 过 @evaluation 的 Go 才进下一阶段

## 阶段 3：IDE 插件 + API（W7-10）

- **IDE 插件**（VS Code/JetBrains）：补全 + 生成，**补全延迟要极低**（@ml-serving 重点）
- **API**：代码生成/补全 API（@backend 流式 FIM 协议）
- 内部试用（50 人），收集反馈

> IDE 补全延迟是体验生死线——补全慢于阈值用户直接关掉。这是代码场景对 @ml-serving 的特殊要求。

## 阶段 4：企业私有化 / 代码 Agent（W10-14）

两条差异化路径（选一或都做）：

- **企业私有化**：模型部署到企业域，用企业私有代码微调 → "懂自家代码库"（数据安全卖点）
- **代码 Agent**：跨文件改 bug、写测试、做 code review（SWE-bench 能力）

> 这是和通用 Copilot 拉开差距的地方。私有化 + Agent 是代码大模型公司真正能建壁垒的方向。

## 阶段 5：商业化（W12+）

- 按 seat 订阅（企业研发团队）
- 私有化部署费（一次性 + 维护）
- 代码 Agent 按任务计费

## 关键决策点（CEO 要拍板）

| 决策 | 选项 | 影响 |
|------|------|------|
| 基座 | 开源微调 / 自训 | 算力成本差数量级 |
| 先做能力 | 补全 / Agent | 决定差异化 vs 红海 |
| 部署 | 云 API / 私有化 | 决定目标客户（中小 vs 大企业） |
| 目标客户 | 个人 / 中小企业 / 大企业 | 决定商业模式和数据壁垒 |

## 风险与对策

| 风险 | 对策 |
|------|------|
| 合规事故（协议/凭证） | @legal 把关 + 100% 过滤 |
| benchmark 污染虚高 | LiveCodeBench + 私有集 |
| 补全延迟过高体验差 | 专门优化（量化+引擎） |
| 只做补全陷入红海 | 押注 Agent/私有化差异化 |
| 大厂 Copilot 降维打击 | 靠私域代码 + 企业定制建壁垒 |
