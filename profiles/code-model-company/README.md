# Profile: 代码大模型公司

> 这是模型工厂的一个**垂直落地配置包**。把通用框架针对"代码大模型公司"场景具体化，给出可参考的上下文、数据策略、评测基准和路线图。
> 用法：`init.sh` 后，参照本目录填充你的 `memory/core/project-context.md`；角色 KPI 和技能使用结合代码场景调整。

## 为什么单独做代码垂直

代码是大模型**变现最快、壁垒最实**的垂直之一：
- **可验证**：代码能跑能测，评测客观（pass@k），不像开放生成那么虚
- **付费意愿强**：企业研发团队愿为提效付费
- **数据壁垒可建**：私有代码库微调、企业内部代码理解
- **生态成熟**：IDE 集成、Copilot 范式已被验证

## 代码大模型公司 vs 通用大模型的关键差异

| 维度 | 通用大模型 | 代码大模型公司（本 profile） |
|------|-----------|---------------------------|
| 数据 | 网页/书籍为主 | **代码为主**（高代码占比），且需**开源协议过滤 + 去凭证** |
| 评测 | 开放生成（LLM judge） | **执行式评测**（pass@k），更客观 |
| 核心能力 | 对话/知识/推理 | 补全(FIM)/生成/调试/代码 Agent/审查 |
| 产品形态 | API + 聊天 | **IDE 插件 + API + 代码 Agent** |
| 商业模式 | 按量 API / 订阅 | IDE 订阅 / 企业私有化 / 按 seat |
| 防污染重点 | benchmark 泄露 | **按时间分窗 benchmark**（LiveCodeBench） |

## 与主仓库的关系

- **不替换**主仓库的通用框架，而是**叠加**一个代码场景的具体化参考
- 主仓库的 15 角色 + 闭环 DAG 不变，本 profile 给出代码场景下的**填充值和侧重**
- 通用 references（训练/对齐/推理/评测/MLOps/数据）依然适用，本 profile 补充代码特有内容

> 这是**一个垂直的参考实现**，不是唯一答案。做法律/医疗/金融大模型可以仿照这个结构做自己的 profile。
