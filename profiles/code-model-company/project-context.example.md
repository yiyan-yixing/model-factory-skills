# 项目上下文示例（代码大模型公司）

> 这是 `memory/core/project-context.md` 的**填充示例**，供 `init.sh` 后参考。
> 示例公司名 "CodeForge" 仅为占位，替换为你的。

## 公司

- **名称**：CodeForge（示例）
- **阶段**：0→1 创立期
- **模式**：大模型公司（数据→模型→服务→商业闭环）
- **方向**：**企业级代码大模型**（代码补全 + 代码生成 + 代码 Agent）

## 产品

- **定位**：面向企业研发团队的代码智能，私域代码理解 + 高质量补全/生成
- **目标用户**：中大型企业研发团队（有私有代码库、重视数据安全）
- **核心假设**：企业愿为"懂自家代码库、数据不出域"的代码助手付费（按 seat 订阅）
- **差异化**：
  - 私有代码库微调（企业内部代码理解，通用 Copilot 做不到）
  - 代码 Agent 能力（不只是补全，能跨文件改 bug、写测试）
  - 数据合规（代码不出企业域）

## 季度 OKR（示例）

- **O**：用企业级代码助手赢得首批付费研发团队
- **KR1**：构建代码训练/评测数据集，去凭证+合规（W3）
- **KR2**：模型在 LiveCodeBench 达标（防污染），pass@1 ≥ 目标线（W6）
- **KR3**：IDE 插件 MVP（补全+生成），内部 50 人试用（W8）
- **KR4**：3 家企业付费试点（按 seat）（W12）

## 明确不做

- ❌ 通用聊天/知识问答大模型（和通用大厂正面竞争）
- ❌ ToC 个人开发者免费工具（先做企业付费，变现路径短）
- ❌ 不做合规的代码训练（用未过滤协议/含凭证的数据 = 法律雷）
- ❌ 只做补全不做 Agent（补全是红海，Agent 是差异化）

## 角色 KPI 调整（代码场景）

| 角色 | 通用 KPI | 代码场景侧重 |
|------|----------|-------------|
| @data-strategy | 数据质量/利用率 | + **开源协议合规率 100%** + 凭证清除率 100% |
| @ml-trainer | 训练可复现 | + 代码占比 + 多语言覆盖 |
| @evaluation | 评测覆盖率 | + **pass@k** + 防污染（时间分窗） |
| @ml-serving | 延迟/成本 | + **补全延迟**（IDE 要求极低延迟） |
| @backend | API 可用率 | + IDE 流式补全协议（FIM） |
| @devrel | 文档/SDK | + **IDE 插件** + 示例代码库 |
